HOW VOID WORKS

01
AES-256-GCM ENCRYPTION
Your message is encrypted using AES-256-GCM, the same algorithm used by governments and militaries worldwide. The key is derived from your password using PBKDF2 with 100,000 iterations and a random salt. Without the exact password, decryption is computationally impossible.
02
LSB STEGANOGRAPHY
The encrypted data is hidden in the Least Significant Bits of the image's pixel values. Each pixel stores 1 bit. Changing the last bit of a color value (e.g., 200 → 201) is invisible to the human eye but encodes your data perfectly.
03
PLAUSIBLE DENIABILITY
Set a decoy message shown when someone enters a wrong password. If you're ever forced to reveal a password, give them the decoy one. They'll see an innocent message and have no way to prove another one exists.
04
ZERO SERVER ARCHITECTURE
VOID runs entirely in your browser using the Web Crypto API. No data is ever sent to any server. No logs. No analytics. No tracking. The code is open source — verify it yourself.
⚠ OPERATIONAL SECURITY
VOID is only as strong as your operational security.
— Send encoded images via Telegram (no compression) or email attachment.
— Never use WhatsApp — it compresses images and destroys hidden data.
— Share your password through a completely separate channel.
— Use VOID CODE mode when you cannot send images.
